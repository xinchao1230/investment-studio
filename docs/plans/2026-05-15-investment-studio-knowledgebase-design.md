# Investment Studio вАФ Two-Layer Knowledge Base Auto-Injection

**Date:** 2026-05-15
**Status:** Approved

## Problem

Stella agent's `knowledgeBase` defaults to empty. When a research chat is bound to a target (e.g. йШњйЗМеЈіеЈі-W 09988.HK), the AI doesn't know about the target's data files (CSVs, reports, PDFs) generated by research-mcp tools or stored manually. Users expect the AI to "know" the current target context.

## Design

### Two-Layer Knowledge Base

| Layer | Path | Scope | Content |
|-------|------|-------|---------|
| Agent-level | `{userData}/portfolio/` | Cross-target | All targets, shared resources, custom `.claude/skills/` |
| Target-level | `{userData}/portfolio/{name}_{code}/` | Single target | Financial data, reports, notes, earnings, models |

### Changes

#### 1. New placeholder: `@KOSMOS_PORTFOLIO_DIR`

Add to `kosmosPlaceholders.ts`. Resolves to `{userData}/portfolio/`.

#### 2. Default Stella knowledgeBase

Set `DEFAULT_CHAT_AGENT_STELLA.knowledgeBase = '@KOSMOS_PORTFOLIO_DIR'` in `profile.ts`. Existing users with empty knowledgeBase get backfilled via profile migration in `profileCacheManager.ts`.

#### 3. System prompt injection (agentChat.ts)

When chat has target binding:
```
**Agent Knowledge Base:** `C:\...\portfolio\`
**Research Target:** йШњйЗМеЈіеЈі-W (09988.HK)
  - Target Directory: `C:\...\portfolio\йШњйЗМеЈіеЈі-W_09988.HK\`
  - All file operations default to this directory
```

When no target (global chat):
```
**Agent Knowledge Base:** `C:\...\portfolio\`
```

Already fixed in prior commit: `hasTargetScope` no longer requires `hasKnowledgeBase`.

### Not Changed

- No new UI panels
- No auto-copy of attached files
- No change to portfolio directory structure
- `.claude/skills/` scanning under knowledgeBase continues to work
